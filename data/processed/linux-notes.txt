To increase the size of the /var directory on an Ubuntu 11 system, you must first determine if it is a separate partition/logical volume or part of the root (/) filesystem, and if you are using Logical Volume Management (LVM). The process is different for each scenario. 
Important Precautions
Backup your data: Modifying disk partitions carries a risk of data loss. Back up all important data before proceeding.
Determine your setup: Run the command df -h and lsblk to see your current disk layout and whether LVM is in use (LVM volumes will appear under /dev/mapper/ names like vg00-var). 
Scenario 1: /var is a separate Logical Volume (Recommended & Most Flexible Method) 
If df -h shows /var as a separate LVM volume (e.g., /dev/mapper/vgname-var), follow these steps. This can often be done online (without unmounting) if the filesystem is ext4 or xfs. 
Check Volume Group (VG) Free Space: Use vgs or vgdisplay <vgname> to check for available free space in the Volume Group that contains your /var logical volume.
Extend the Logical Volume (LV):
To add a specific amount of space (e.g., +5GB):
bash
sudo lvextend -L +5G /dev/mapper/vgname-var
Use code with caution.

To use all available free space in the VG:
bash
sudo lvextend -l +100%FREE /dev/mapper/vgname-var
Use code with caution.

Replace vgname-var with your actual logical volume name.
Resize the Filesystem: After extending the LV, you must resize the filesystem to use the new space.
For ext4 or ext3 filesystems:
bash
sudo resize2fs /dev/mapper/vgname-var
Use code with caution.

For xfs filesystems, the command is xfs_growfs (which requires the package to be installed first).
Verify: Check the new size with df -h. 
Scenario 2: /var is a directory within the Root (/) Partition
If /var is part of your root filesystem (which is likely if you don't see a separate mount point for it in df -h), you need to extend the root partition. This often requires booting from a Live CD/USB (like a GParted Live CD or Ubuntu installation media) because you cannot resize a mounted root partition. 
Boot from Live Media: Boot your system from a GParted Live CD or a separate Ubuntu installation medium.
Use a Partition Editor: Use a graphical tool like GParted or command-line tools like fdisk or cfdisk to resize the partition.
Ensure there is unallocated space adjacent to the partition you want to extend. You may need to move other partitions (like swap) to free up adjacent space.
Extend the partition to fill the desired unallocated space.
Apply changes within the tool and reboot the system into your main Ubuntu installation.
Resize the Filesystem: Once back in your main system, run the filesystem resize command (if it didn't happen automatically):
bash
sudo resize2fs /dev/sdaX 
Use code with caution.

(Replace /dev/sdaX with your actual root partition name, e.g., /dev/sda1).
Verify: Check the new size with df -h. 
Alternative: Clearing Space Temporarily 
If you need immediate space to perform the above steps, you can:
Clear the apt cache: sudo apt-get clean.
Check /var/log for very large log files and manage them. 
For systems where LVM is not an option, you can also move data-intensive subdirectories (like /var/lib/mysql or /var/log) to a new, larger disk/partition and use a mount point or symlink.
